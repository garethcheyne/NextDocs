## :star: Core Concepts

### Documentation Structure

Your documentation is organized hierarchically:

- **Categories**: Top-level sections (e.g., Getting Started, User Guide)
- **Pages**: Individual markdown files within categories
- **Navigation**: Controlled by `_meta.json` files

### Content Format

All documentation is written in Markdown with these enhancements:

- GitHub Flavored Markdown (GFM)
- Inline icons with custom syntax
- Mermaid diagram support
- Automatic link transformation

## :bulb: Tips for Success

1. **Keep it Simple**: Write clear, concise documentation
2. **Use Examples**: Code samples and diagrams help readers understand
3. **Stay Organized**: Use folders and `_meta.json` to structure content
4. **Test Your Links**: Always verify links work after syncing

## :question: Common Questions

**How do I add a new page?**
Create a new `.md` file and add its reference to `_meta.json`.

**How do I change page order?**
Reorder the items in your `_meta.json` file.

**Can I nest categories?**
Yes! Create subfolders with their own `_meta.json` files.

**How do I add images?**
Place images in your repository and reference them: `![Alt text](./image.png)`
